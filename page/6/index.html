一些感想

本事无所谓的，但是提起的次数多了，自己渐渐也就难以忘却。
“从前从前，有个人爱你很久”周杰伦的歌是最适合的。适合这个时候，鄙人的一位朋友，近来在某个交友平台上获得了自己所谓的爱情。确实我觉得也确实像是爱情，至于说为什么不确定地说呢?那是因为我是局外人，看不得一些内幕，这种东西如人饮水，冷暖自知。只能祝福罢了。
我也回想起来自己的过去与现在，老实讲，我是无感的，这些一切的一切，我都知道是什么滋味，自己内心也渐渐封闭，对这种东西的需求，并不能算强烈。爱情当然很好，但是对于我来说，学业的成功更加重要，这本来就是我最看重的。从前便是如此。之后也会将如此。

那就需要说明一下：爱情确实难得，这对每个人来说都是一种奢侈品，易碎，需要精心维护。可是我已经渐渐没有这个心思了。这大概就是人开始老去吧！所以还是要多和年轻一点的人多互动。

今天的工作完成也没有达到预期，我的具体分析如下：

  • 第一是晚上和朋友说话多了，没有控制好时间。
  • 对于遇到难题的难度估计不足，没有完成。
  • 自己的动力不足，毕竟ddl还是比较远

一往无前
如何一往无前？这是个问题，生活中有太多的让人分心的因素，还是需要让这些因素远离自己，这样才能聚精会神，完成任务。

不要为之前的感情消耗情绪

Posted on 2024-05-08 Edited on 2025-09-29

人的经历是有限的，只能用于有限的方面，可以说现在每时每刻都有事物来争夺人们宝贵的注意力。人们可能隔几个小时就要刷一刷微
博，看一看小红书。每天写完文章，工作结束之后，相信不少人都身心俱疲，这时候一些意志力强大的会去干一些能够继续提升自己的事情，比如健身，学习，研究；意志不坚定的人可能刷一刷电视剧就洗洗睡了。
这些注意力是宝贵的，人的精力也是有限的。
这就要求我们不要将精力浪费在那些没有意义
的事情上。这些精力很大程度上就是你的情
绪，注意力。相信大家会有着这种经历，考试考砸之后，会消极颓废，不想学习，明明自己需要努力学习，但是没有了向前的动力；或
者，今天发生了一件非常影响自己情绪的事
情，然后整个人一天都没有欲望去干别的事
情，只是在逃避不想去面对这件糟糕的事情。
这种现象，我愿称之为情绪消耗之后的能动性不足。干一件事情是需要消耗人的情绪，不如耐心，注意力，心情。人不可能无限制地处理面对这些事情。
以前的感情便是一种人们不值得为之浪费情绪的事物。之前的感情是一段破碎亲密关系的余响，是剔除了很不好的回忆之后，片段式的，
具有欺骗性质的一种印象。随着时间的流逝，
回忆起之前的感情大多是亲密的甜蜜的，给人以非常美好的假象。这常常在人观察之前的旧感情时添上一层不全面的滤镜，会使人觉得美好。
如果真有这样甜美，圆满，那么为什么还会有
分手这样不美满的结局？答案其实在你心里。
当事人心理之前的感情并不真实，是自己大脑无意识的趋利避害的结果。换言之，大脑欺骗了自己。
之前的感情的种种不愉快的细节被人无意识地
忽略了。
与之前的感情纠缠，最为明显的表现便是和之前的对象纠缠不清，藕断丝连。这种行为没有意义。
藕断丝连最好的结果就是复合，但这最好的结果，是当事人想要追求的吗？当事人心里的那个彩虹泡泡般的幻想究竟和现实是不是一样？
这是个很大的问题。不排除有着复合之后的幸福美满，但是这样不符合我们的精神，我们讨论的是一种概率，一种在数据量较大时才会稳定的概率。个例并不能说明概率是错误的；并且这种幸福是建立在双方都需要做出让步和改变，否则同样的事情会以另一种形式再次发
生，戳破当事人心中的彩虹泡泡，最后留下一地鸡毛。改变自己的难度并不是那么简单，想一想自己能不能坚持每天健身，没有一天懈
怠，就可以大致了解到难度。
复合之后呢？当事人会发现自己想象的浪漫，
美满会有；同时也会有不和谐，尴尬，难看，
不愉快。没有一种东西是纯粹的，爱情更是美好的浪漫与无休止的不满妥协的混合体。当事人之前的选择就是不接受这种混合体，之后大概率的选择是什么，相信自己心里也有大概的预估。
所以本人原则上不接受不建议不提倡为之前的
感情再消耗情绪。

往事

过桥

我把她的名字在三生石上刻了二百六十遍。当她在奈何桥上看到我的时候，却眼神冰冷，不似从前看我笑靥如花，她不留一丝牵挂，直直地过桥去了，只有口中还喃喃地说些什么，后来问了问孟婆，我才知道，她口中一直在说“你不爱我了”。

桥前

第一次遇见她的时候，我还是一个被贬在此处的仙人，全无前世的记忆，“奈何桥前长流水，直叫伊人短相思”她一个人走过来想要忘却前世的种种，业障太重不得过桥再生，唯有忘却前世的情缘。她默默地走过来，缓缓地举起那碗孟婆汤，颤颤巍巍，凝视良久，在孟婆的催促下，送到嘴边，却仍旧回头望了望。转头之后心灰意冷，无奈摇首叹息。含入口中。不知孟婆汤是何滋味，余光所及，眼中却突放光芒。她看见了我。我不禁心中疑惑，这个凡间女子为何对我有所
羁绊，这桥前忘却今生，转向来世投胎是仙家规矩。我走过前去，对她说；“来者可知要饮孟婆汤”。她还没有擦干的眼泪在月光的照耀下分外晶莹剔透，可谓“寒蝉凄切，对长亭晚，骤雨初歇”般的凄冷，空明。她弱弱地说到：“你都不愿意记得我吗？”我很是疑惑，仙家官人怎会
与民女相识，何来相识一说。她看我不回答，声音更加低落，细弱游丝，其
中更增加了几分哀怨，：“苏大才子有一篇《江城子》，里面有一段唱词，我想为你唱一段，可否？”我看见她近乎哀求的预期，弱柳扶风，不由惹人怜爱，我本是天神下凡，但看到此景不由心神不宁，只想让她快快过桥去。但是心中却又有几分触动，便随了她的愿望。

起舞

今夜月凉如水，月光碎了一地，连空气也凝固住。一种庄严的凝重压在我身上，让我喘不过气来，我不知道自己是怎么回事，也无法可想，无径可知。自己坐罢，静静的准备看着她
起舞。远处传来悠悠笛声，断空绝响。她着素服，起微微调，慢慢摆好了姿势。
佳人起舞，素衣微澜，长袖舞动，似流水潺潺,口中唱着：“十年生死两茫茫，不思量自难忘，
今日相逢不相识，泪满面，冷若霜”。舞姿婉转，似弱柳扶风，一张一弛，尽显佳人本色。
我不知为什么心中有一种难以遏制的冲动，脑中飞速运转，这修改过的苏词，意有所指。难不成指的是我？可是我与此女子并不相识，为何对我纠缠不休？心中无限疑惑。又不经想到自己好像忘却了什么，忘却了什么十分重要的事情，仔细想时脑壳十分痛楚，好像有一千尊佛像围着你，念着经文，不止地说道经文的真谛，告诫佛法无边，勿泛俗念。

往事重现

我冷汗不止地滴落，心中无名的泛起了无尽的痛楚，一些前世的片段慢慢闪现，我看到了一个熟悉的身影，慢慢来看，仔细端详尽然是她的曼姿在月下舞动，似梦似幻，亦真亦假，幻影重重中又看到现在的她在月下独舞。幻境和现实交织，在这个时刻汇集。恍惚间白光闪过，前世的记忆如潮水般融入，我看到了我们的相识相知，相恋相爱，一起宣
誓，结下不朽誓言。我看到我快离去时，她哭得梨花带雨，声泪俱下。眼中尽是不舍。不想此生再见竟然只是路人。

过桥

她跳完了舞，看了看我，没有回响。径直向奈何桥走去，回头看我眉头紧锁，却没
有反应，迈下了那沉重的一步。奈何桥，人奈何，往日今生晨水去，唯留相思悲情曲。笛声消逝，空响九环。她终究还是过桥了。我在痛苦中挣扎，上天降下神罚，我此刻只能眼睁睁看着她过桥，忘记与我的一切。我只能痛苦地凝噎，纵使青筋暴起，接近全力，也只能失能般抽搐。一下一下向前蠕动。慢悠悠的，慢悠悠的，又想起了往昔她的笑容。恍惚间看见她在桥对岸等我，看见她在向
我挥手，一边大喊我的名字。我不停地向前爬去，爬到桥上，一直向前爬，
企图追上她的影子。桥上猛然间怪石嶙峋，我在锋利的石头中伤痕累累，血迹一直桥的起点。我精疲力竭，大口喘气，爬一段便要歇息好久。直到意识不清，仅仅凭借残存的意识，机械地向前爬去。不知过了多久，不止爬了多远，我模模糊糊看见，清清楚楚看见她就在我身边和我一起向对岸走去。
一月三十一日夜（大概是2023年1月31日，2025年8月13日校正）

博客模板功能介绍

Posted on 2020-07-11 Edited on 2025-09-29

在此之前我有写过一篇如何使用我博客模板的文章 请先看这篇基础集成，这篇文章主要讲的是如何使用和修改我的博客主题，文章内容含以下几点：

博客特性

  • 适配电脑、手机、平板等各屏幕
  • 响应式设计
  • 个性化头像
  • 每篇文章自动添加打赏功能
  • 支持Disqus、livere评论系统
  • 支持站点总数访问统计，每篇文章访问统计
  • 支持文章自动生成目录
  • 支持标签分类
  • 支持代码高亮
  • 支持文章H1、H2、H3、H4标题样式多样化
  • 支持多种三方社交icon展示，能从博客直接跳转到自己的三方社交主页
  • 支持三方社交分享(facebook、twitter)

博客主要模块介绍

_config.yml

_config.yml 是博客的配置文件，整个站点的信息都在这修改，想要把我的模板改成你自己的也需要修改_config.yml

重要字段说明

  • enableToc: 是否开启文章自动生成目录，设置为false文章不会自动生成目录
  • comment/livere: livere评论系统，支持微信、qq、微博、豆瓣、twitter等登录后可以直接评论
  • comment/disqus: disqus评论系统，支持facebook、twitter等登录后可以直接评论
  • social/weibo、github、zhihu、jianshu等: 个人站底部展示的微博等三方社交按钮，点击后直接跳转到个人微博或其他社交主页
  • baidu/id: 百度统计，用来统计你个人站点的用户访问情况
  • ga/id: google统计，用来统计你个人站点的用户访问情况

_config.yml 文件除以上字段还有一些可以自行修改，例如title之类的字段

_posts

_posts 目录是用来存放文章的目录，写新文章，直接放在这个目录即可

使用博客模板时，请把博客自带的文章给去掉，如果想使用博客自带的文章请 注明出处。

自定义页面

about.md、support.md 等为自定义页面，如果你想添加自动以页面可以直接复制about.md 文件修改文件名和里面的内容即可。

如果需要在导航显示你新增的页面，直接在_config.yml 文件的nav字段中添加你新页面配置即可

修改说明

如果要修改博客模板信息建议只修改_config.yml 文件内容和 _posts 里面的文章信息。因为博客模板一直在更新迭代，改动多了以免你后期更新博客模板的时候不方便。

如果你想改动模板的样式又想继续更新迭代博客模板，你可以提交在github上提交pull request 或者直接给我发邮件建议改成什么样，如果你的提议确实可以，我会采纳的，并且非常感谢你的建议。

博客迭代信息请看ReleaseNode

什么是 Markdown

Markdown 是一种方便记忆、书写的纯文本标记语言，用户可以使用这些标记符号以最小的输入代价生成极富表现力的文档：如您正在阅读的这篇文章。它使用简单的符号标记不同的标题，分割不同的段落，粗体 或者 斜体 某些文字.

很多产品的文档也是用markdown编写的，并且以“README.MD”的文件名保存在软件的目录下面。

Jekyll搭建个人博客

Posted on 2016-10-14 Edited on 2025-09-29

　之前写了一篇HEXO搭建个人博客的教程获得了很好评，尤其是在简书上目前已经累积了10W+的阅读量了，也有好心的读者主动给我打赏，在此感谢。

　如果你看过我的文章会发现我现在的博客样式跟之前是有很大的区别的，之前我也是使用 HEXO 搭建的博客，后来发现使用 HEXO 在多台电脑上发布博客，操作起来并不是那么方便，果断就转到了 Jekyll 上，接下来我会讲如何使用 Jekyll 搭建博客，博客模板效果。

介绍

　Jekyll 是一个简单的博客形态的静态站点生产机器。它有一个模版目录，其中包含原始文本格式的文档，通过 Markdown （或者 Textile） 以及 Liquid 转化成一个完整的可发布的静态网站，你可以发布在任何你喜爱的服务器上。Jekyll 也可以运行在 GitHub Page 上，也就是说，你可以使用 GitHub 的服务来搭建你的项目页面、博客或者网站，而且是完全免费的

　使用 Jekyll 搭建博客之前要确认下本机环境，Git 环境（用于部署到远端）、Ruby 环境（Jekyll 是基于 Ruby 开发的）、包管理器 RubyGems
　　如果你是 Mac 用户，你就需要安装 Xcode 和 Command-Line Tools了。下载方式 Preferences → Downloads → Components。

　　Jekyll 是一个免费的简单静态网页生成工具，可以配合第三方服务例如： Disqus（评论）、多说(评论) 以及分享 等等扩展功能，Jekyll 可以直接部署在 Github（国外） 或 Coding（国内） 上，可以绑定自己的域名。Jekyll中文文档、Jekyll英文文档、Jekyll主题列表。

### 修改成你自己的博客

>* 如果你想使用我的模板请把 _posts/ 目录下的文章都去掉。
>* 修改 _config.yml 文件里面的内容为你自己的。

然后使用 git push 到你自己的仓库里面去，检查你远端仓库，在浏览器输入 username.github.io 就会发现，你有一个漂亮的主题模板了。      


### 为什么要是用 Jekyll

使用了 Jekyll 你会发现如果你想使用多台电脑发博客都很方便，只要把远端 github 仓库里的博客 clone 下来，写文章后再提交就可以了，Hexo 由于远端提交的是静态网页，所有无法直接写 Markdown 的文章。如果你想看 Hexo 搭建博客，可以看看我的另一篇[HEXO搭建个人博客](http://leopardpan.cn/2015/08/HEXO%E6%90%AD%E5%BB%BA%E4%B8%AA%E4%BA%BA%E5%8D%9A%E5%AE%A2/)的教程。

如果你在搭建博客遇到问题，可以在[原文博客](http://leopardpan.cn/2016/10/jekyll_tutorials1/)的评论里给我提问。

后面会继续介绍，在我的博客基础上，如何修改成你自己喜欢的 Style，欢迎继续关注我博客的更新。
